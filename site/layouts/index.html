Question 2

What is your favorite thing about providing technical support?

My favorite thing about providing technical support is the process of figuring out the root of a problem and working towards finding the solution from there. I love knowing that I helped solve a problem that was causing someone frustration, because knowing that that person is feeling better helps me to feel accomplished.

Question 3

What did you think of our service during the time you used it? Provide either some constructive criticism or some points that impressed you. Be honest! “It sucked” isn’t a wrong answer unless you don’t elaborate and provide some constructive criticism ;)

I really like Netlify! As you’ll see in my answer below, one of the challenges I faced during this exercise was getting a bit more familiar with how GitHub works (since I’ve mainly used SourceTree for my work), however once I had a better understanding of how it’s all integrated I found that Netlify is a pretty easy tool to use! The continuous deployment makes committing changes SO much easier, as it takes absolutely no time at all to see the work you put in go live on the site.
One piece of constructive criticism I have is to offer the option to refresh deployment next to the Site Settings and Domain Settings buttons on the Overview page. I did find that I always had to refresh the entire page every time I wanted a new build to appear, so having a button there that the user can click might make it that much faster. However, maybe I still haven’t fully grasped how the continuous deployment works and there’s a button elsewhere that I’m missing. Otherwise, I found it to be really easy and the fact that it’s just a quick three step process makes it all the much better!

Question 4

Talk about how you made your site and why you chose the tools you did. Briefly explain a challenge you experienced in setting up this site and how you solved it.

I created my site by using the Hugo template listed on the provided StaticGen link in the exercise. I then attempted to create the landing page myself, but was not pleased with the way it looked so I took a template from a website I use often: w3schools. I implemented the Start Page CSS Template that they provided into the index.html file in the Site -> Layouts folder, then switched out the text to represent the questions asked in the exercise and the answers I provided (and removed the icons they had on that template). It still doesn’t look exactly how I would like, but it’s vastly better than it was originally!
In all honesty, I never really knew how to build a website as I was always testing on sites that were already built (or I would create websites via a CMS site like WordPress). Therefore, the biggest challenge I had was getting a better understanding of how a website’s code is stored and which file to change in order to change the landing page.
Another challenge I faced was the aforementioned challenge with GitHub – though I use GitHub through SourceTree at my current job, I haven’t used it as an individual site so learning how to navigate the website was a bit difficult at first, but I think I got the hang of it!

Question 6

Why do you think SSL/HTTPS is important?

In my opinion, SSL/HTTPS is important for three main reasons: 1) protecting the security and privacy of a website’s users, 2) ensuring trust between a user and that website (there’s something to be said about seeing that little green lock symbol in the URL bar), and 3) keeping that website as usable/findable as possible. Not only is SSL/HTTPS important to protect the privacy and security of a website’s users, but it’s also important to develop trust between the user and that website. As someone who goes to many daifferent websites daily, there’s something comforting to be said about the little green lock next to the website’s URL.
